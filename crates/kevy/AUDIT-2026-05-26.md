### Coverage gap explanation (cmd.rs 44%, dispatch.rs 43%)

cmd.rs holds the per-command implementations (cmd_get, cmd_set, cmd_hset,
… ~60 commands). dispatch.rs routes verb → cmd impl. The uncovered
branches are:

1. Argument-count error paths for many commands (e.g. "HSET requires
   ≥3 args" branch)
2. WRONGTYPE error paths for type-incompatible commands
3. Less-common command edge cases (LSET out-of-range, ZADD NX/XX
   semantics, etc.)

These ARE exercised in production by:
- `bench/compat3.sh` — 94 commands diffed against valkey 9.1 and
  redis 7.4 reply-by-reply (NOT measured by llvm-cov since shell-driven)
- `bench/compat.sh` — single-server diff harness

But that test suite does NOT generate cov metrics for the workspace-cov
gate. To raise the in-Rust cov: add a `tests/cmd_matrix.rs` that hits
every command at least once with both happy + arity-error + wrong-type
inputs, measured via cargo-llvm-cov.

**This is the highest-priority follow-up for this audit.** Not blocking
production (Phase A stones + sharded suite + compat3 catch real bugs),
but the cement bar formally isn't cleared.
