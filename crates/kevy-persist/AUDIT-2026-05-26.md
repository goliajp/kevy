## Performance / hot path

This cement is NOT on the per-command hot path. AOF append happens for
write commands, but at human-perceptible interval (Fsync::EverySec by
default ‚Äî fsync once per second, batches in between). Snapshot save is
explicit SAVE/BGSAVE. Load happens only on startup.

Per CEMENT-AUDIT ¬ß4, BUDGETS.md is required only for hot-path cement.
**Skipped for kevy-persist** (correctly per spec).

## Notes / future considerations

- AOF format is RESP-shape (replay parses commands back via
  `kevy_resp::parse_command`). Couples to RESP wire format, not just to
  kevy-store. Could split AOF off as its own cement if AOF becomes
  fundamentally different from snapshot ‚Äî currently both are intertwined
  (Store fully drives both).
- Snapshot magic is `kevySnap` (8 bytes); version is 1. Pre-publish
  should document the format and bump policy if we publish.
