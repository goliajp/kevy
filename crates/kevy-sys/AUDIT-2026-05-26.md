## FFI vs man page spot check

Sampled 4 critical extern fns + matched against the platform man page
they should marshal to:

### `socket(domain: c_int, ty: c_int, protocol: c_int) -> c_int`
- **POSIX**: `int socket(int domain, int type, int protocol)` → ✅ matches (c_int ~ int).

### `bind(fd: c_int, addr: *const c_void, addrlen: u32) -> c_int`
- **POSIX**: `int bind(int sockfd, const struct sockaddr *addr, socklen_t addrlen)` → ✅
  matches; `addrlen: u32` because socklen_t is u32 on linux+macos.

### `epoll_wait(epfd, events, maxevents, timeout) -> c_int` (Linux)
- **Linux man**: `int epoll_wait(int epfd, struct epoll_event *events, int maxevents, int timeout)` → ✅ matches.
- Note: `EpollEvent` is `#[repr(packed)]` on x86_64 (matches kernel ABI) and naturally aligned on other arches per the cfg branch — documented in the source comment at lib.rs:124.

### `madvise(addr: *mut c_void, length: usize, advice: c_int) -> c_int` (Linux)
- **Linux man**: `int madvise(void *addr, size_t length, int advice)` → ✅ matches
  (`size_t` ≈ `usize`).
- The advice constant `MADV_HUGEPAGE = 14` matches `asm-generic/mman-common.h`.

Comprehensive sig review of all ~16 extern fns is a one-time pre-publish
activity; the above spot-check confirms the discipline. **TODO before
publishing**: write `PLATFORMS.md` documenting tested target list and any
non-standard ABI assumptions (e.g. the packed EpollEvent x86_64 quirk).
