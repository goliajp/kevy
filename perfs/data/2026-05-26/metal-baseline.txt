v0.metal-1 — measurement foundation baselines (lx64, io_uring, single shard).
All host-loopback, server core 0, client cores 1-15.

== per-core CPU (cache-hot, single key) ==
GET -c50 -P256 ≈ 5.9M/core  (~169 ns/cmd ≈ ~590 cyc @3.5GHz) — see
  local-reply-bypass-ab.txt. This is the headline but cache-hot (1 key, L1).

== MEMORY WALL (the dimension the single-key bench hid) ==
bench/metal_keyspace.sh — preload N keys (verified: -r 1e6 -n 2e6 → DBSIZE
864,734, RSS 164 MB ≈ 190 B/key), GET random over the keyspace:
  keyspace        GET rps     vs 1-key
  1               4,756,781   —          (L1-hot)
  100,000         3,387,832   -29%       (~8 MB, > L2)
  1,000,000       2,713,366   -43%       (~80 MB, > L3)
  10,000,000      2,283,757   -52%       (~1 GB, deep DRAM + TLB)
=> Real large-keyspace single-shard GET ≈ 2.3M/core (DRAM-bound), HALF the
   cache-hot headline. THE biggest untapped lever. Levers: value/key inlining
   (kill the value pointer-chase 2nd cache miss), software prefetch of the next
   key's bucket in the batch, cache-conscious bucket layout, hugepages (TLB).

== cross-core ring (kevy-ring SPSC) — bench/../bench_ring on lx64 (x86) ==
  same-thread push+pop : ~1 ns
  cross-thread SPSC    : cap=256 9.5 ns/item (104.8M/s), cap=1024 6.1 ns/item (165M/s)
=> The cross-core transport is CHEAP on x86 (~6-9 ns/item, batched even less).
   So the cross-core ring is NOT a major cost here — deprioritize v0.metal-3.
   (mac M-series showed ~63 ns/item — coherency differs wildly; lx64 is authoritative.)

== mem footprint ==
~190 B/key (key ~16B + value 3B + bucket/entry overhead) at 864k keys / 164 MB.

== binary size ==
release (stripped) ~0.4 MB (profile has strip=true); profiling build w/ debug
syms ~668 KB. (size axis baseline = stripped shipping binary.)

== DATA-INFORMED PRIORITY (reorders v0.metal cold checkpoints) ==
1. MEMORY WALL (-52%) — value inlining + prefetch + hugepages   [was metal-6 → now next]
2. zero-alloc hot path (~10% malloc/cfree)                       [metal-2]
3. parse (~13.5%) + dispatch                                     [metal-4/5]
4. cross-core (ring only 6-9ns/item — low) / footprint / size    [metal-3/9]
