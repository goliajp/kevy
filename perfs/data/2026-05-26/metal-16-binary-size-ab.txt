→ ≈ 50 KB (~7%) of the binary is std's backtrace machinery (gimli,
  addr2line, miniz_oxide, OnceCell init paths). On stable Rust this is
  pulled in even with `panic = "abort"` — std unconditionally links
  symbol-resolution for `RUST_BACKTRACE=1`. Removing requires nightly
  `-Z build-std -Z build-std-features=panic_immediate_abort` (charter
  prefers stable; out of scope).


== opt-level trade-off experiment ==

Added `[profile.release-min]` inheriting `release` with `opt-level` swapped.
Built kevy with each, measured 3-run medians via metal_keyspace.sh.

  opt-level   binary       1-key       100k       1M         10M       Δ size vs base
  3 (base)    672 KB       4.54M       4.23M     3.72M       3.53M       —
  "s"         572 KB       4.30M       3.76M     3.47M       3.27M       -15%
  "z"         551 KB       3.32M       3.24M     2.97M       2.82M       -18%

  Δ rps vs base:
  opt=s   1-key -5.4%   100k -11.0%   1M -6.7%   10M -7.3%
  opt=z   1-key -27%    100k -23%     1M -20%    10M -20%

→ opt-level=s costs **-5 to -11%** rps for **-15%** binary. Above the
  perf-WIN gate's 5% paired-regression threshold (100k -11% > 5%).
- opt-level=z is a disaster: -20 to -27% rps for -18% binary.

→ KEEP opt-level=3 on `release`. The `release-min` profile (opt-level="s")
  is preserved in Cargo.toml as an alternate users can opt into when
  size matters more than throughput (e.g., embedded / container image
  layers). Default ships are opt=3.


== Other angles considered (rejected) ==

- `panic = "abort"`: already set; std's backtrace machinery is the
  remaining ~50 KB but can't be stripped on stable.
- `strip = true`: already set.
- `lto = "fat"`: already set.
- `codegen-units = 1`: already set.
- Monomorphisation trim: `KevyMap<K, V>` instantiates as
  `KevyMap<SmallBytes, Entry>`, `KevyMap<SmallBytes, Vec<u8>>`, and
  `KevyMap<SmallBytes, f64>`. Three instantiations is the minimum for
  the type-safety story; merging via type erasure (Box<dyn ...>) would
  hurt the dispatch chain perf metal-8 just won back.
- `#[cold]` on rare paths: shrinks icache footprint, not binary bytes.

== Judgement: KEEP base release at opt=3; SHIP release-min as opt-in ==

  perf axis: unchanged (no rebuild of release binary).
  mem axis : unchanged.
  size axis: base 672 KB unchanged; release-min (opt-in) is 572 KB at
             -15% size cost of -5 to -11% rps. Trade-off documented for
             callers; the project's default ship stays perf-first.

→ The size axis is fundamentally constrained by stable Rust's
  unconditional std backtrace machinery (~50 KB / 7% of binary). Real
  size wins beyond opt-level tuning need nightly `-Z build-std` with
  `panic_immediate_abort` — out of charter without major rationale.

→ This stone (well, "checkpoint" — not a stone in the mailrs ARCHITECTURE
  sense) closes the size axis as exhausted on stable. Phase E next is
  metal-17 cross-core arena (lowest remaining ceiling).
