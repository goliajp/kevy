# 19x rise from -P1 to -P256 => per-command syscall/reactor overhead DOMINATES,
# not command CPU (which is ~120ns; at -P256 single-shard is 1.8M = 555ns/cmd,
# so >75% is non-command-CPU: io_uring SQE submit/reap + reactor per-iteration).
#
# Key A/B insight: -P256 (same 4-shard server) reaches 7.2M > -P16's 3.33M, so
# -P16 4-shard is SERVER-BOUND (the 12-core client can clearly drive 7.2M). That
# makes "io_uring 4sh, client 12c, GET -c50 -P16" a clean server-bound A/B config
# for reactor/syscall optimizations.
#
# Biggest lever: at -P16, ~16x more read SQEs than at -P256 (one read SQE per
# ~16 commands vs ~256). io_uring MULTISHOT recv + a provided-buffer ring would
# cut read-SQE submit/re-arm to ~once per connection (the kernel re-fires and
# picks a buffer), attacking exactly this overhead. ~2x headroom at -P16.
