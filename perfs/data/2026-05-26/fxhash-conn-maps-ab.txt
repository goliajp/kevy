  -P64  GET: develop 3,421,630 ; 3,434,276   feature 4,283,716 ; 4,267,455   +25%

=> single-shard GET ceiling 3.77M -> ~4.78M/core (+25%), ~1.9x valkey's TOTAL.

Found by `perf record` (the user enabled perf on lx64; paranoid relaxed then
restored): the single-shard reactor spent ~17.6% of CPU in std SipHash
(core::hash::BuildHasher::hash_one 11.4% + DefaultHasher::write 6.2%) — the
per-command lookups of conns: HashMap<u64,Conn> (twice/command), fd_to_conn, and
the io_uring UringConn map. Internal integer-keyed maps, no DoS surface, so
SipHash was pure overhead. FxHashMap (write_u64 specialized, ~1ns vs ~15ns) gave
the +25%. Unlike the command-CPU reserve (system-neutral), this is on the reactor
path so it moves throughput. Overturns the earlier "conn-map hash ~3ns,
negligible" assumption — measure-first put it at the top.
